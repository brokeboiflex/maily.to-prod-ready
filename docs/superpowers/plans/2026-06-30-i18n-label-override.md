# i18n Label-Override Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Make every user-facing string in `@maily-to/core` overridable by a caller-supplied, complete language dictionary, with English defaults and zero coupling to any i18n framework.

**Architecture:** Ship a flat, dot-namespaced English dictionary (`defaultLabels`) plus a tiny translator `t(key, vars?)`. `Editor` resolves `labels = props.labels ?? defaultLabels` once, memoizes `t = createTranslator(labels)`, and hands `t` to its three string consumers: the default-block builder (`getDefaultBlocks(t)`), the Placeholder/slash-command extensions (via `defaultExtensions({ …, t })`), and every chrome component (via `MailyContext`). Resolution is a reference swap, never a merge.

**Tech Stack:** React 18/19, TypeScript, TipTap/ProseMirror, Vitest 3 (already a root devDep), tsup. Design spec: `docs/superpowers/specs/2026-06-30-i18n-label-override-design.md`.

## Global Constraints

- **Framework-agnostic.** No `next-intl`/`react-i18next`/etc. Data in, string out.
- **Full language, no partial overrides.** `MailyLabels = Record<LabelKey, string>` (NOT `Partial`). A consumer object omitting any key is a **compile error naming the key**. Never merge a partial object against defaults.
- **English by default.** `labels` omitted ⇒ `defaultLabels` used verbatim. No behavior change for existing consumers who don't pass `labels`.
- **Merge-free resolution:** `props.labels ?? defaultLabels`; `t` memoized on the `labels` reference; lookups are O(1) property access.
- **No backward compat (decided):** block exports (`text`, `button`, …) and `PlaceholderExtension` become **factory-only** `(t) => …`. The old plain-object/instance exports are dropped from `@maily-to/core/blocks` and `@maily-to/core/extensions`. Bump `packages/core` minor version and document the break.
- **`searchTerms` stay English** and are NOT in the dictionary (fuzzy-match keys, never displayed; overridable only via the `blocks` prop). **Inserted seed content** (e.g. footer "UNSUBSCRIBE") is out of scope.
- **`packages/render` is untouched.**
- **Plain Tailwind tokens only; never hand-edit `registry/**`** — it is generated by `pnpm registry:build` from `packages/*/src`.
- **Testing:** lightweight pure unit tests for the i18n core + `tsc --noEmit` for the completeness guarantee. No DOM/RTL harness.

```ts
// packages/core/src/editor/i18n/translate.ts
import { defaultLabels, type LabelKey, type MailyLabels } from './default-labels';

export type TranslateFn = (
  key: LabelKey,
  vars?: Record<string, string | number>
) => string;

/** Replace `{token}` occurrences with values from `vars`. Leaves unknown tokens intact. */
function interpolate(
  str: string,
  vars?: Record<string, string | number>
): string {
  if (!vars) {
    return str;
  }
  return str.replace(/\{(\w+)\}/g, (_, k) =>
    k in vars ? String(vars[k]) : `{${k}}`
  );
}

export function createTranslator(labels: MailyLabels): TranslateFn {
  return (key, vars) => interpolate(labels[key], vars);
}

/** Stable English translator for default params and out-of-provider fallbacks. */
export const englishTranslator: TranslateFn = createTranslator(defaultLabels);
```
